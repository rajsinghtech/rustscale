### Key algorithms (port from Go, no need to read Go sources)

**`tailscale_4to6`**: Take `Tailscale4To6Range().ip` as 16-byte array. Copy `v4.octets()[1..4]`
(octets 2-4: e.g. for `100.64.0.5` → `[64,0,5]`) into `ret[13..16]`. Return as `Ipv6Addr`.

**`tailscale_6to4`**: Verify ip is within `tailscale_4to6_range()`. Extract bytes `[13..16]`
from v6, prepend `100` → `[4]byte{100, v6[13], v6[14], v6[15]}` → `Ipv4Addr`.

**`map_via`**: Copy first 8 bytes of via range (`fd7a:115c:a1e0:b1a`). Write `site_id` as
big-endian u32 at `a[8..12]`. Copy v4 address bytes at `a[12..16]`. Prefix length =
`v4.bits() + 64 + 32`.

**`unmap_via`**: If `ip` in `tailscale_via_range()`, extract `a[12..16]` as Ipv4Addr.
Else return `ip` unchanged.

**`is_tailscale_ip`**: v4 check = `cgnat_range().contains(ip) && !chrome_os_vm_range().contains(ip)`.
v6 check = `tailscale_ula_range().contains(ip)`.

**`sort_prefixes`**: Sort by ip addr bytes then by bits ascending (use `std::cmp::Ordering`).

**`is_exit_route`**: `p == all_ipv4() || p == all_ipv6()`. Use the `==` operator on IpPrefix
(compare ip + bits).

### Collection primitives

```rust
use std::sync::atomic::{AtomicU64, Ordering};
use std::sync::{Arc, Mutex};
use std::collections::HashMap;

#[derive(Clone, Default)]
pub struct SockStats {
    inner: Arc<Mutex<SockStatsInner>>,
}

struct SockStatsInner {
    counters: HashMap<Label, LabelCounters>,
    current_interface_cellular: bool,
}

#[derive(Clone, Copy, Debug, Default, serde::Serialize)]
pub struct LabelCounters {
    pub tx_bytes: u64,
    pub rx_bytes: u64,
}

impl SockStats {
    pub fn new() -> Self { ... }
    /// Get a handle to increment counters for a label (create on first access).
    pub fn label_handle(&self, label: Label) -> LabelHandle { ... }
    /// Snapshot all counters (for JSON serialization).
    pub fn snapshot(&self) -> HashMap<Label, LabelCounters> { ... }
    /// Whether the current active interface is cellular.
    pub fn current_interface_cellular(&self) -> bool { ... }
    pub fn set_current_interface_cellular(&self, v: bool) { ... }
}

/// Cheap clone — shares the same atomic counters.
#[derive(Clone)]
pub struct LabelHandle {
    tx: Arc<AtomicU64>,
    rx: Arc<AtomicU64>,
}
impl LabelHandle {
    pub fn record_tx(&self, n: usize) { self.tx.fetch_add(n as u64, Ordering::Relaxed); }
    pub fn record_rx(&self, n: usize) { self.rx.fetch_add(n as u64, Ordering::Relaxed); }
    pub fn counters(&self) -> LabelCounters { ... }
}
```

**No platform-specific socktrace/TCP connection info** (Go uses a modified Go runtime for
that). Instead, use **manual instrumentation**: each socket send/recv path calls
`handle.record_tx(n)` / `handle.record_rx(n)`. This is less magical but works on std Rust.

## 4. IPNAUTH — Unix peer credential checking

### Goal

Replace the current socket-0600-only auth model in the LocalAPI accept path with Go's
`ipn/ipnauth` ConnIdentity model: extract Unix peer credentials (uid/pid) from each
incoming connection and enforce read-write vs read-only access.

### Current state

`safesocket/src/unix.rs` — `listen()` creates a socket with 0o666 perms on
peer-cred platforms, or 0o600 elsewhere. `platform_uses_peer_creds()` returns true
on linux/macos/freebsd/solaris/illumos. But the LocalAPI accept loop
(`crates/tsnet/src/localapi.rs:spawn_localapi`) does NOT extract or check peer credentials.

### What to implement

**`crates/safesocket/src/peercred.rs`** — Unix peer credential extraction helpers
(platform-specific, `#[cfg]` blocks):

```rust
/// Peer credentials extracted from a connected Unix socket.
#[derive(Clone, Debug)]
pub struct PeerCreds {
    pub uid: u32,
    pub pid: u32,
}

/// Get peer credentials from a connected UnixStream.
///
/// Linux: SO_PEERCRED → struct ucred (pid, uid, gid)
/// macOS: LOCAL_PEERCRED → struct xucred (version, uid, gid) + getpeereid()
///
/// Returns None when credentials are not available (e.g. peer is a local TCP
/// connection on the macOS sameuserproof path).
#[cfg(any(target_os = "linux", target_os = "macos", target_os = "freebsd"))]
pub fn get_peer_creds(stream: &tokio::net::UnixStream) -> Option<PeerCreds> { ... }

#[cfg(not(any(target_os = "linux", target_os = "macos", target_os = "freebsd")))]
pub fn get_peer_creds(stream: &tokio::net::UnixStream) -> Option<PeerCreds> { None }
```

Implementation details:

**Linux** (`target_os = "linux"`):
- Use `libc::getsockopt(fd, SOL_SOCKET, SO_PEERCRED, &mut ucred)`.
- `struct ucred` has fields `pid: libc::pid_t`, `uid: libc::uid_t`, `gid: libc::gid_t`.
- Get fd via `stream.as_raw_fd()`.

**macOS** (`target_os = "macos"`):
- Use `libc::getpeereid(fd, &mut uid, &mut gid)` via `libc`.

### ConnIdentity model

Add to `crates/safesocket/src/peercred.rs`:

```rust
/// Represents the identity of the connecting peer.
/// Ports Go's `ipn/ipnauth.ConnIdentity`.
#[derive(Clone, Debug)]
pub struct ConnIdentity {
    pub uid: Option<u32>,
    pub pid: Option<u32>,
    pub is_unix_sock: bool,
}
impl ConnIdentity {
    pub fn from_stream(stream: &tokio::net::UnixStream) -> Self {
        let creds = get_peer_creds(stream);
        Self {
            uid: creds.as_ref().map(|c| c.uid),
            pid: creds.as_ref().map(|c| c.pid),
            is_unix_sock: true,
        }
    }

    /// Whether this connection should be granted full read-write access.
    /// Mirrors Go's `ConnIdentity.IsReadonlyConn` inverted logic.
    ///
    /// Read-write if:
    /// - No creds → depends on policy (default read-only)
    /// - uid is 0 (root) OR
    /// - uid matches daemon's uid (same user, non-root) OR
    /// - uid matches operator_uid (if set)
    pub fn is_readwrite(&self, daemon_uid: u32, operator_uid: Option<u32>) -> bool {
        let Some(uid) = self.uid else { return false; };
        uid == 0 || uid == daemon_uid || operator_uid.is_some_and(|op| uid == op)
    }
}
```

### Enforcing in the LocalAPI accept path

In `crates/tsnet/src/localapi.rs`, change `spawn_localapi`:

```rust

    state: Arc<LocalApiState>,
    socket_path: PathBuf,
) -> Option<LocalApiHandle> {
    let listener = rustscale_safesocket::listen(&socket_path).ok()?;
    // --- NEW: get daemon uid once ---

```

Actually, the cleanest approach: compute daemon_uid once at spawn time using
`std::os::unix::process::parent_id` won't work. Instead:

```rust
#[cfg(unix)]
fn daemon_uid() -> u32 {
    unsafe { libc::getuid() }
}
```

Then in the accept loop, before dispatching, extract ConnIdentity and pass it into
the handler chain so each handler can check `is_readwrite`. Currently the handler
chain doesn't distinguish between read-write and read-only callers.

### Minimal integration approach (no full Go-Actor model)

Keep it simple: add a `conn_identity: Option<ConnIdentity>` field to the per-connection
handler context. The `handle_connection` function already has access to `stream` (which
is a `UnixStream`). Extract creds right after accept:

```rust
match listener.accept().await {
    Ok(stream) => {
        let peer_identity = rustscale_safesocket::peercred::ConnIdentity::from_stream(&stream);
        let state = state.clone();
        tokio::spawn(async move {
            if let Err(e) = handle_connection(stream, &state, peer_identity).await {
                eprintln!("localapi: connection error: {e}");
            }
        });
    }
    ...
}
```

Then in each handler that modifies state (PATCH prefs, POST start, login-interactive,
logout, shutdown, file-put, serve-config write, set-expiry-sooner), check:

```rust
fn require_readwrite(identity: &ConnIdentity) -> Result<(), LocalApiError> {
    let daemon_uid = unsafe { libc::getuid() };
    if !identity.is_readwrite(daemon_uid, None) {
        return Err(LocalApiError::AccessDenied);
    }
    Ok(())
}
```

Return HTTP 403 with body `{"error": "access denied"}` on failed check.

### What NOT to implement

- Windows named-pipe token checking (separate task, Windows uses `ImpersonateNamedPipeClient`)
- macOS sameuserproof TCP path (not a Unix socket, creds not available)
- Full Go Actor model (WindowsToken, WithPolicyChecks, profile-level access control)
- `operator_uid` configuration parsing (hardcode to `None` for now)
- macOS admin group / QNAP administrators group membership check

### Acceptance criteria

- `tools/check.sh safesocket` green
- `PeerCreds::get_peer_creds()` compiles and returns `Some` on linux/macos (test manually)
- LocalAPI starts and accepts connections
- Read-write endpoints reject callers who connect as a different UID
- Tests verify: same-UID → RW, different-UID → 403 on mutating endpoints
- Read-only endpoints (status, health, metrics, whois) return 200 regardless of UID
