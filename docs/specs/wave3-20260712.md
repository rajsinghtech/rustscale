**NEW: `crates/tsdial/src/dns_map.rs`** — MagicDNS name→IP resolution:

Key facts (do NOT read Go sources):

- `dnsMap` is `HashMap<String, IpAddr>` mapping lowercased hostname (no trailing dot)
  to IP address.
- Built from `NetworkMap` on each update: for the self node, add `name` and
  `shortname` (stripped tailnet domain suffix) → first IP; skip self if v4-only
  when peer has no v4. For each peer with name → first addr. For each
  `ExtraRecord` with empty Type → add.
- `canon_map_key(s)`: lowercase + trim trailing `.`.
- `resolve(ctx, network, addr)`: parse host:port; if host is literal IP return
  `SocketAddr`; if host found in map return `SocketAddr(ip, port)`; otherwise
  `Err(Unresolved)`.
- `split_host_port(addr) → (host, u16)`: parse `host:port` or `[host]:port`.
- Already implemented pattern: `rustscale_dnscache::Resolver` exists for system
  DNS; `dns_map::DnsMap` is a *different* cache for MagicDNS → tailnet IP.

**`system_dial` internals:**
- Uses `netns::dial_tcp(host, port)` — the existing function at `crates/netns/src/lib.rs:79`
  that wraps proxy detection, SOCKS5 fallback, and platform-specific bind-to-interface.
- Tracks connections in `active_sys_conns` (`HashMap<u64, Arc<tokio::net::TcpStream>>`).
- On link change (via `netmon`), closes all tracked connections (wraps each in a
  `SysConn` that on `Close()` removes itself from tracking).

**`user_dial` internals:**
- Resolves addr via 3-tier: (1) `DnsMap::resolve_memory` (MagicDNS), (2) system
  resolver, (3) exit-node DoH proxy (if `exit_dns_doh` set).
- For single-result / UDP: `dial_one_user`.
- For multi-result TCP: happy-eyeballs with 300ms fallback delay (`race_dial_user`).
- `dial_one_user` decision: if `use_netstack_for_ip(ip)` → netstack dial; else if
  route table shows tailscale-route → `get_peer_dialer()` (plain `TcpStream::connect`,
  no bind); else → `system_dial`.

**`dial_peer_api`:** plain `TcpStream::connect` to peer IP:port — no netns, no proxy.

**`peer_api_http_client`:** `reqwest::Client::builder().no_proxy().build()` wrapping
`dial_peer_api` as the connector (not yet possible with reqwest on nightly rustls;
fallback: build an `http::Client` over a custom connector, or document as TODO and
expose `peer_api_transport()` returning a `tokio_rustls::TlsConnector`).

### (c) Key facts from Go (coders should NOT read Go sources)

- The three dial paths serve different purposes: `SystemDial` for outbound-to-internet
  (control plane, DERPs, upstream DNS) — netns-bound, connection-tracked for link-change
  teardown. `UserDial` for user-initiated traffic (SOCKS, tsnet.Dial, DNS forwarder) —
  route-aware, happy-eyeballs, NOT tracked. `dialPeerAPI` for peer-to-peer — plain TCP
  no bind.
- `UserDialPlan` is a TOCTOU-aware advisory: returns the resolved addr and whether
  it would go via Tailscale. It's a separate call because DNS resolution is expensive.
- DNS resolution order: MagicDNS → system → exit-node DoH.
- The `routes` atomic bypasses the `use_netstack_for_ip` closure for the route decision.
  Stored as `std::sync::atomic::AtomicPtr<...>` or (simpler) an `ArcSwap<RouteTable>`.
- On link change: close all `active_sys_conns` (system connections). User connections
  are NOT tracked and NOT closed.
- `DnsMap` rebuilds from netmap each time `SetNetMap` is called; the Go version stores
  it in the `dns` field behind `mu`.
- `StdDialer` does NOT exist in the Go package. The plain fallback is `&net.Dialer{}`
  (no netns config).

/// Returns a `RootCertStore` containing the two baked-in ISRG roots (X1 and X2).
/// Lazily initialized on first call.
pub fn get() -> &'static rustls::RootCertStore {
    static STORE: OnceLock<rustls::RootCertStore> = OnceLock::new();
    STORE.get_or_init(|| {
        let mut store = rustls::RootCertStore::empty();
        store.add_parsable_certificates(
            rustls_pemfile::certs(&mut ISRG_ROOT_X1_PEM.as_bytes())
                .chain(rustls_pemfile::certs(&mut ISRG_ROOT_X2_PEM.as_bytes()))
                .collect::<Result<Vec<_>, _>>()
                .expect("baked ISRG root PEMs are valid"),
        );
        store
    })
}

/// Build a `RootCertStore` that concatenates webpki roots, optional extra
/// certs, and baked ISRG roots — the 3-tier strategy as a single store.
pub fn combined_root_store(extra_certs: Option<&[Vec<u8>]>) -> rustls::RootCertStore {
    let mut store = rustls::RootCertStore::empty();
    store.roots.extend(webpki_roots::TLS_SERVER_ROOTS.iter().cloned());
    if let Some(extras) = extra_certs {
        for der in extras {
            store.add_parsable_certificates(std::iter::once(der.clone()))
                .expect("extra root cert should be valid DER");
        }
    }
    store.roots.extend(get().roots.iter().cloned());
    store
}

## T3: PROXYMAP + PORTLIST (`crates/proxymap`, `crates/portlist`)

### (a) Goal

Two small crates:

1. **proxymap** — ephemeral `(protocol, localhost IP:port) → Tailscale IP` registry.
   Used by netstack's TCP handler to register proxied connections so that WhoIs can
   attribute them. Three methods: `register`, `unregister`, `whois_ipport` (with
   retry 0/10/20/50/100ms).

2. **portlist** — periodic enumeration of listening TCP/UDP ports on the host
   (macOS: `netstat` + `lsof` for process names; Linux: `/proc/net/tcp{,6}`, `/proc/net/udp{,6}`
   + `/proc/*/fd` for process names). Reports results as `Vec<tailcfg::Service>`.

**NEW: `crates/portlist/src/linux.rs`:**
- Opens `/proc/net/tcp`, `/proc/net/tcp6`, `/proc/net/udp`, `/proc/net/udp6` at init.
- Parses hex fields: skip header row, parse local addr field (`XXYYZZWW:PPPP` where
  WW.XX.YY.ZZ is little-endian IPv4, PPPP is hex port).
- Filters: only listening (remote = `00000000:0000` or `00000000000000000000000000000000:0000`).
- Filters out localhost (v4: `0100007F:`, v6: `00000000000000000000000001000000:`).
- Builds inode key `socket:[{inode}]` → `Port` entry.
- Second pass: walks `/proc/*/fd/`, calls `readlink()` on each looking for
  `socket:[{inode}]` match, reads `/proc/{pid}/cmdline` for process name.
- Use `std::fs::read_dir("/proc")` to enumerate PIDs. Filter numeric entries only.
- Poll interval: 1 second.

**NEW: `crates/portlist/src/macos.rs`:**
- First pass: spawn `netstat -na`, parse TCPv4/v6/UDPv4/v6 columns, filter LISTEN
  state, extract port numbers.
- Second pass (only if new ports appeared): spawn `lsof -F -n -P -O -S2 -T -i4 -i6`,
  parse field-based protocol (p=c PID, c=command, P=protocol, n=address).
- Poll interval: 5 seconds.
- Cache known ports in a `HashMap<(proto, port), PortMeta>` with `keep` flag.
- On macOS sandbox (App Store), `lsof` may fail silently — cache the failure in
  an `AtomicBool` and skip process-name fetching thereafter.

**NEW: `crates/portlist/src/stub.rs`:**
- Returns empty list for unsupported platforms.
