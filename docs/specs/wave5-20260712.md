## 1. TAILSCALE SSH PHASE 1 — SERVER-SIDE SESSION ORCHESTRATOR

### (a) Goal

The `crates/ssh/` (`rustscale-ssh`) crate has the full building blocks:
`SshServer`/`SshHandler` (russh protocol), `auth::eval_ssh_policy`, `Session`,
`Incubator` (shell spawn with privilege drop), `hostkeys`, `env`, `recording`.
The `tsnet::listen_ssh()` wires them into a netstack TCP listener that yields
`Session` values from an `SshListener::accept()`.

What's **missing** is the session orchestrator — the equivalent of Go's
`ssh/tailssh/tailssh.go:handleSession()` — which:
1. Resolves the local user (getpwnam_r / CPassword on Windows)
2. Sets up Go-style SSH env vars (SSH_CLIENT, SSH_CONNECTION, etc.)
3. Opens a PTY via `openpty(3)` when requested
4. Calls `Incubator::spawn()` to launch the login shell
5. Pumps data bidirectionally: SSH channel ←→ shell stdio  
6. Forwards signals and window-size changes from SSH to the child
7. Optionally records the session via `SessionRecorder` (asciicast v2)
8. Reports exit status back to the SSH client

Phase 1 delivers a `run_session` function that `tsnet::SshListener::accept()`
users call in a per-session task. Deferred to later phases: session recording
upload (PeerAPI stream), HoldAndDelegate, check/verification URLs, SFTP.

### (b) Files to create/modify

#### `crates/ssh/src/session_handler.rs` — NEW

The core `run_session` orchestrator:

```rust
//! SSH session orchestrator — ports Go's handleSession (tailssh.go:handleSession).
//! Takes a Session, resolves the local user, spawns the shell, and pumps I/O.

/// Errors from session handling.
#[derive(Debug, thiserror::Error)]
pub enum SessionHandlerError {
    #[error("local user resolution failed: {0}")]
    LocalUser(String),
    #[error("PTY allocation failed: {0}")]
    Pty(String),
    #[error("incubator error: {0}")]
    Incubator(#[from] IncubatorError),
    #[error("IO error: {0}")]
    Io(#[from] std::io::Error),
    #[error("channel closed")]
    ChannelClosed,
}

/// Run a session to completion: resolve user, spawn shell, pump I/O.
///
/// Mirrors Go's `handleSession` (tailssh.go ~1100). Returns the exit code.
///
/// # Arguments
/// * `session` — the accepted SSH session (from SshListener::accept)
/// * `rec_config` — optional recording configuration (None = no recording)
pub async fn run_session(
    session: Session,
    rec_config: Option<RecordingConfig>,
) -> Result<i32, SessionHandlerError>
```

Implementation steps (port of Go's `handleSession`):

1. **Resolve local user**: call `get_local_user(session.user())` which wraps
   `libc::getpwnam_r` (unix) to get uid/gid/groups/home_dir/shell. Return
   `SessionHandlerError::LocalUser` if the user doesn't exist. Mirrors Go's
   `lookupUser` in tailssh.go.

2. **Build env vars**: start with `session.environ()`, add standard SSH vars:
   - `SSH_CLIENT="{src_ip} {src_port} {dst_port}"`
   - `SSH_CONNECTION="{src_ip} {src_port} {dst_ip} {dst_port}"`
   - `SSH_TTY={tty_name}` (if PTY allocated, else empty)
   - `LOGNAME`/`USER` = local_user, `HOME` = home_dir, `SHELL` = login_shell
   - `PATH` = default (e.g. `/usr/local/bin:/usr/bin:/bin` if not set)
   - `TERM` = from PTY if allocated
   Mirrors Go's `envForSSH` (tailssh.go).

3. **Allocate PTY** (if `session.pty().is_some()`):
   - Call `openpty(3)` via `libc::openpty` to get master+slave fds
   - Set `TIOCSWINSZ` on the slave from `session.pty().window`
   - Store the master fd for signal forwarding and I/O pumping
   - Convert the slave fd into a tokio file handle for shell stdio
   - Store the tty_name (ttyname_r on the slave fd) for SSH_TTY
   Mirrors Go's `setupPTY` (tailssh.go).

4. **Build `IncubatorArgs`** with resolved user info, env, session command.

5. **Create `SessionRecorder`** if `rec_config` is Some — factory is
   `RecordingConfig::new_recorder(session.peer_node(), session.user())`
   from `crate::recording`.

6. **Spawn shell** via `Incubator::spawn()`:
   - If PTY: dup slave fd onto stdin/stdout/stderr before spawn (using
     `unsafe { CommandExt::pre_exec }` to dup2 the slave fd)
   - If no PTY: use the default pipes from Incubator

7. **Pump I/O** concurrently (tokio::select! / join!):
   - **stdin pump**: read from `session` (AsyncRead) → write to shell stdin
   - **stdout pump**: read from shell stdout → write to `session` (AsyncWrite)
   - **stderr pump**: read from shell stderr → write to session via
     `handle.extended_data(channel_id, EXTENDED_DATA_STDERR, data)` — this
     requires storing the `russh::server::Handle` in Session (it's already
     in `SessionInit.handle`).
   - **signal handler**: receive signals from `session.handle` (russh
     forwards them via `SshHandler::signal`) — but signals are handled
     per-connection via the `SshHandler`. Actually in the current design,
     the SshHandler receives signals and they're stored in the handler
     state. The Session struct doesn't have a signal receiver. Options:
     (a) Add a `signal_rx: mpsc::Receiver<Sig>` to SessionInit/Session.
     (b) Use a shared atomic for the latest signal.
     For Phase 1, the simplest is to have the session handler task check
     a signal channel in the pump select loop. Add `signal_rx` to
     `SessionInit` and wire it in `SshHandler` where `signal()` is called.

   **Simplified pump architecture** (avoiding tokio files on raw fds):
   ```rust
   // If PTY: use the master fd as a readable+writable AsyncFd
   // If pipe: use tokio::process::ChildStdout etc.
   let result = tokio::select! {
       // Read from SSH channel, write to shell stdin
       v = pump_stdin(&mut session, shell_stdin) => v,
       // Read from shell stdout, write to SSH channel (data)
       v = pump_stdout(shell_stdout, &mut session) => v,
       // Read from shell stderr, write to SSH channel (extended_data)
       v = pump_stderr(shell_stderr, session_handle, channel_id) => v,
       // Forward SSH signals to child process
       v = forward_signals(signal_rx, &mut spawned) => v,
       // Forward window changes to PTY (TIOCSWINSZ ioctl)
       v = forward_window_changes(window_change_rx, pty_master_fd) => v,
   };
   ```

8. **Wait for child** to exit, return exit code.

9. **Recording**: pass recorded data through the `SessionRecorder` (the
   Session's AsyncWrite already calls the recorder; the recorder flush
   should happen on session exit).

### (c) Key facts distilled from Go (ssh/tailssh/tailssh.go + ssh.go)

1. **Authentication flow**: Go uses `conn.sshPolicy()` to read `netMap.SSHPolicy`,
   calls `evalSSHPolicy` (already ported in `crates/ssh/src/auth.rs`). The
   existing russh auth flow matches Go: `auth_none`/`auth_password`/`auth_publickey`
   all call `tailscale_auth`.

2. **handleSession pipeline**:
   - `determineUser()` → getpwnam SSH user (honours user-mapping from rule)
   - `setupEnv()` → SSH_CLIENT, SSH_CONNECTION, SSH_TTY, LOGNAME, HOME, SHELL
   - `setupPty()` → openpty, TIOCSWINSZ, grantpt/unlockpt
   - `startWithPTY()` or `startWithPipes()` → spawn shell
   - Shell is ALWAYS the user's login shell (from passwd), NOT `$SHELL`
   - `forwardSignals()` → goroutine reading SIGINT/SIGTERM/SIGHUP from a channel
   - `handleSessionPost()` → pump loop, wait, recording finalize

3. **SSH env vars format**:
   - `SSH_CLIENT` = `"{clientIP} {clientPort} {serverPort}"` — matches OpenSSH
   - `SSH_CONNECTION` = `"{clientIP} {clientPort} {serverIP} {serverPort}"`
   - `SSH_TTY` = PTY device name (e.g. `/dev/pts/5`)

4. **PTY via openpty**: Go uses `pty.Open()` from `github.com/creack/pty`
   which wraps `openpty(3)`. In Rust use `libc::openpty`:
   ```rust
   let mut master: std::os::raw::c_int = 0;
   let mut slave: std::os::raw::c_int = 0;
   let result = unsafe { libc::openpty(&mut master, &mut slave, ptr::null_mut(), ptr::null_mut(), ptr::null_mut()) };
   ```
   Then `set_winsize(master, &winsize)` via `TIOCSWINSZ` ioctl.

5. **Shell as child**: Go re-execs tailscaled as `be-child ssh` subprocess.
   For Rust, the simpler approach (already implemented in `Incubator`) spawns
   the login shell directly via `tokio::process::Command` with uid/gid
   privilege drop via `pre_exec`. This matches Go's fallback path when
   `tailscaledPath` is empty.

6. **Signal mapping**:
   - `russh::Sig::Int` → SIGINT
   - `russh::Sig::Term` → SIGTERM
   - `russh::Sig::Hup` → SIGHUP
   - `russh::Sig::Quit` → SIGQUIT
   - `russh::Sig::Kill` → SIGKILL
   Go sends these to the child process group.

7. **Recording**: Go's `recorder` wraps asciicast v2 writer. The Rust
   `SessionRecorder` already exists in `crate::recording`. Recording output
   is already wired in `Session::poll_write`. The recorder should be flushed
   on session exit.

8. **SSH_HostKeys**: Populated from Hostinfo in `crates/tsnet/src/hostinfo.rs`
   line 297-298. The `ssh_host_keys` field in `RuntimeConfig` is set
   externally. The `host_key_from_node_key` generates an ed25519 SSH host key
   from the node key (deterministic, so survives restarts).

## 3. PEER MTU — DF SOCKET PLUMBING

### (a) Goal

The PMTUD probe logic (multi-size ping bursts, probe-size tracking per
endpoint, `WIRE_MTUS_TO_PROBE`) is already implemented in
`crates/magicsock/src/lib.rs`. What's missing is the platform-specific

the magicsock UDP sockets, plus the `UpdatePMTUD` orchestration function

than returning EMSGSIZE, so the probe logic never detects MTU limitations.

//! IPPROTO_IP + IP_MTU_DISCOVER = IP_PMTUDISC_DO (enable) / IP_PMTUDISC_DONT (disable)
//! IPPROTO_IPV6 + IPV6_MTU_DISCOVER = same values

use crate::pmtud::platform::conn_control;

/// Enable/disable DF on a UDP socket for the given network (udp4/udp6).
pub(crate) fn set_dont_fragment(
    socket: &std::net::UdpSocket,
    network: &str,
    enable: bool,
) -> Result<(), anyhow::Error> {
    let opt_arg = if enable {
        libc::IP_PMTUDISC_DO
    } else {
        libc::IP_PMTUDISC_DONT
    };
    let ip_proto = if network == "udp4" { libc::IPPROTO_IP } else { libc::IPPROTO_IPV6 };
    let opt = if network == "udp4" { libc::IP_MTU_DISCOVER } else { libc::IPV6_MTU_DISCOVER };
    conn_control(socket, network, |fd| {
        let ret = unsafe { libc::setsockopt(fd, ip_proto, opt, &opt_arg as *const _ as *const _, std::mem::size_of_val(&opt_arg) as u32) };
        if ret != 0 { /* capture errno */ }
    })?;
    Ok(())
}

### (c) Key facts distilled from Go (peermtu*.go)

1. **Socket option values**:
   - **Linux**: `IP_MTU_DISCOVER` (setsockopt level `IPPROTO_IP`) with value
     `IP_PMTUDISC_DO` (2) to enable DF, `IP_PMTUDISC_DONT` (0) to disable.
     Same for IPv6 with `IPV6_MTU_DISCOVER` at level `IPPROTO_IPV6`.
   - **Darwin**: `IP_DONTFRAG` (10) at level `IPPROTO_IP` with value 1/0.
     `IPV6_DONTFRAG` (55) at level `IPPROTO_IPV6` with value 1/0.

2. **connControl pattern**: Go uses `sc.SyscallConn().Control(fn)` to get
   the raw fd. In Rust, `UdpSocket` implements `AsRawFd` — use that for
   the `setsockopt` call. No need for `SyscallConn` equivalent for simple
   integer options.

3. **errUnsupportedConnType**: When the pconn is not a real UDP socket
   (e.g. in tests), return a sentinel error. In Rust, this happens when
   the socket is wrapped or the fd is invalid.

4. **UpdatePMTUD decision logic**:
   - Read desired state from `ShouldPMTUD()` (envknob → control knob → false)
   - If same as current, return
   - Call `setDontFragment("udp4", enable)` and `setDontFragment("udp6", enable)`
   - If at least one succeeded and neither had a non-ignorable failure → keep
   - Otherwise → force-disable, revert socket opts
   - If changed → `resetEndpointStates()` to trigger re-probe

5. **resetEndpointStates**: Clears each endpoint's `peer_mtu` and sets
   `last_full_ping` to zero so next discovery ping burst includes all sizes.

6. **EMSGSIZE suppression**: When `pmtuShouldLogDiscoTxErr` returns false,
   the send error is not logged (avoids log spam from normal PMTUD activity).

## 4. NETLOG — NETWORK FLOW LOGGING

### (a) Goal

Port Go's `wgengine/netlog/` (477 + 218 = 695 lines) + `types/netlogtype/`
(157 lines) — per-connection network traffic logging aggregated over 5s
windows, uploaded via logtail.

Architecture: a `Logger` registers a `ConnectionCounter` callback into the
packet filter (virtual traffic, tun) and magicsock (physical traffic, raw
UDP). Every 5s the accumulated `Connection`+`Counts` pairs are serialized
as a `netlogtype::Message` JSON blob and sent to logtail collection
`"tailtraffic.log.tailscale.io"`.

Phase 1: crate structure, types, logger with virtual traffic only (tun
path), no recording upload (logtail integration exists in `crates/logtail`).
Phase 2: physical traffic from magicsock, exit-node anonymization,
`ReconfigRoutes` support.

### (c) Key facts distilled from Go

1. **Two traffic sources**:
   - **Virtual** (tun): what applications inside the tailnet see — tailnet IP
     ↔ tailnet IP or tailnet IP ↔ subnet IP. Source is always from the
     perspective of the current node.
   - **Physical** (magicsock): what hits the wire — UDP encapsulation of
     WireGuard packets. Source is always a tailscale IP, destination is a
     non-tailscale IP (the other node's endpoint).

2. **Classification** determines which of 4 buckets traffic goes into:
   - `virtualTraffic`: src is self, dst is known peer in tailnet
   - `subnetTraffic`: src is self, dst is within subnet routes
   - `exitTraffic`: src is self, dst is NOT in tailnet AND NOT in subnet
   - `unknownTraffic`: unclassified (default)

3. **5s poll**: `time.AfterFunc(pollPeriod, flush)`. Flush sends the record
   to a background processing goroutine. The flush timer avoids continuous
   goroutine wakeups. In Rust, use `tokio::time::interval` or a one-shot
   timer that resets after each flush.

4. **Max log size**: 256 KiB. If the record's estimated JSON size exceeds
   this, flush early. Go uses `jsonLen()` upper-bound estimates per node
   and `MaxConnectionCountsJSONSize` per connection.

5. **Logtail collection**: `"tailtraffic.log.tailscale.io"`. The log message
   is a single JSON blob per 5s window. Logtail handles retry, batching,
   compression. Use `rustscale_logtail::LogTail::write()`.

6. **Exit traffic anonymization**: By default, exit traffic scrubs port
   numbers and non-Tailscale IP addresses to protect privacy. Controlled
   by `anonymizeExitTraffic` parameter (opposite of `logExitFlowEnabled`).

7. **Node dedup**: `seenNodes` map tracks which node IPs have been
   encountered to avoid duplicate Node entries in the message.

8. **ConnectionCounter signature**:
   ```go
   type ConnectionCounter func(proto ipproto.Proto, src, dst netip.AddrPort, packets, bytes int, recv bool)
   ```
   Where `recv=true` means the packet was received (Rx), `false` means
   transmitted (Tx).
